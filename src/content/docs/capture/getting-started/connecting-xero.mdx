Each client in Capture connects to a single Xero organisation. Once connected, Capture syncs the data it needs to process and publish documents.

## Connecting a client

<Steps>
    1. Navigate to your client dashboard in Capture and click **Add client**

    2. You'll be redirected to Xero's authorisation screen, and may need to log in

    3. Select the client you want to connect and click **Allow access**

    4. You'll be redirected back to Capture. The connection is now active and data sync begins automatically

</Steps>

## What permissions does Capture request?

Capture requests access to:

- **Contacts:** To match suppliers and customers on your documents, and create new contacts that are not yet in Xero
- **Accounting:** To read your chart of accounts, tax rates, and currencies, and to create bills and invoices
- **Files:** To attach original documents to bills and invoices

Capture does not request access to data outside of what's needed for document processing.

## What syncs after connecting?

Once connected, Capture syncs the following data from Xero:

| Data                     | Purpose                                                                                                                           |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| **Contacts**             | Match suppliers and customers to extracted document data                                                                          |
| **Chart of accounts**    | Assign account codes to line items                                                                                                |
| **Tax rates**            | Apply the correct tax treatment to line items                                                                                     |
| **Currencies**           | Set the document currency                                                                                                         |
| **Tracking categories**  | Assign tracking categories to line items                                                                                          |
| **Organisation details** | Determine base currency, multi-currency support, tax registration status, and other details needed to process documents correctly |

## Connection status

Each client shows one of four connection statuses on the **Clients** page and on the client settings page:

| Status                | Meaning                                                                                            |
| --------------------- | -------------------------------------------------------------------------------------------------- |
| **Active**            | Connected and syncing. Otto can publish bills and invoices to Xero.                                |
| **Pending setup**     | Connected, but setup isn't finished. Finish setup before submitting documents.                     |
| **Disconnected**      | The connection has been removed, either by you in Capture or by revoking Capture's access in Xero. |
| **Connection failed** | Capture tried to sync or publish and Xero rejected the connection. Reconnect to resume.            |

An **Active** client shows its Xero organisation name on the client settings page. You can check the synced data is up to date from the last sync time there.

<Aside type="tip">
  If synced data looks outdated, you can trigger a manual refresh from the
  client configuration page.
</Aside>

## Disconnecting

You can disconnect a client from Xero at any time from the client settings. Disconnecting:

- Stops all data syncing between Capture and Xero
- Prevents new documents from being published to Xero
- Does **not** delete any documents already in Capture
- Does **not** remove any bills or invoices already published to Xero

## Reconnecting

If a client's connection drops, Otto can no longer publish to Xero or refresh synced data, so you'll need to reconnect. A connection drops in one of two ways:

- **You disconnect it in Capture.** The client moves to **Disconnected**.
- **Capture's access is revoked from inside Xero.** Otto checks each client against Xero once a day and mirrors the change, so the client moves to **Disconnected** too. If a sync or publish reaches the revoked connection before that daily check, the client shows **Connection failed** instead.

Either way, organisation admins get a notification with a reconnect link.

To reconnect, open the client settings and click **Reconnect to Xero**. Clients that need attention also show a **Reconnect** action in the **Needs setup** list on the **Clients** page. Reconnecting triggers a fresh data sync, and Otto resumes publishing. Your existing documents and synced data are kept throughout.
