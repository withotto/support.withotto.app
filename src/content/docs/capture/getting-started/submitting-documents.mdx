There are two ways to submit documents to Capture: uploading through the web interface, or forwarding them by email.

## Upload via the web

From your client's document inbox, you can upload files by dragging and dropping them onto the page, or by clicking the upload button and selecting files.

<Image
  src={processingTab}
  alt="The client view showing a document that was emailed being extracted by Otto"
/>

After uploading:

1. The file is scanned for malware
2. Once the scan passes, the document enters the processing queue
3. Otto processes the document (typically within a few minutes)
4. The document appears in your review queue

## Email submission

Each client has a unique email address (set up in [client configuration](/capture/getting-started/configuring-clients)). Forward receipts and invoices to this address, and Capture will automatically extract and process the attachments.

How email submission works:

- Capture processes all supported file attachments from the email
- For emails without attachments, Capture will use the email contents
- Otto reads the email body to understand special cases, such as where the client has provided details of what the purchase was for
- Duplicate emails are automatically detected and skipped
- Each attachment becomes a separate document in Capture

<Aside type="tip">
  Email submission is particularly useful when your clients can forward receipts
  directly from their inbox. Set up an email forwarding rule to automate this
  completely.
</Aside>

## What happens after submission

Every document goes through the same processing pipeline:

1. **Malware scan:** Checked for security threats. Infected files are quarantined and won't be processed
2. **Triage:** Otto classifies the document type (bill or sales invoice) and verifies it's a genuine financial document. Documents that can't be classified need manual classification
3. **Extraction:** Otto extracts structured data: supplier name, invoice number, dates, amounts, line items, tax, and currency
4. **Validation:** Line item amounts are checked against totals for arithmetic consistency. Mismatches are flagged for review but don't prevent processing
5. **Contact matching:** The extracted supplier or customer name is matched against your existing contacts (exact match first, then fuzzy matching)
6. **Ready for review:** The document appears in your review queue with its extracted data

<Aside>
  If Otto detects multiple documents in a single image (for example, two
  receipts photographed together), the document will be flagged for manual
  splitting
</Aside>

## Document statuses

| Status                   | Meaning                                                 |
| ------------------------ | ------------------------------------------------------- |
| **Pending scan**         | Waiting for malware scan to complete                    |
| **Queued**               | In the processing queue                                 |
| **Processing**           | Otto is working on it                                   |
| **Triage failed**        | Otto couldn't confirm it's a genuine financial document |
| **Extraction failed**    | Otto couldn't extract data from the document            |
| **Needs review**         | Extraction complete, ready for you to check             |
| **Needs classification** | Otto couldn't determine the document type               |
| **Needs manual split**   | Multiple documents detected in one image                |
| **Reviewed**             | You've reviewed and confirmed the data                  |
| **Publishing**           | Being sent to your accounting platform                  |
| **Published**            | Successfully published to your accounting platform      |
| **Publish failed**       | Failed to publish. Check the error and retry            |
| **Quarantined**          | Malware detected, file has been quarantined             |
| **Discarded**            | Manually discarded by a user                            |
